# Automated Service Recovery Solution

## Overview

Designed and implemented a PowerShell-based automated service recovery solution for a critical business application environment.

The solution was developed to address recurring service outages caused by high CPU utilization, which impacted application availability and required repeated manual intervention from the IT Support team.

## Business Problem

High CPU utilization (85–95%) caused critical backend services to become unavailable, preventing front-end users from accessing a business-critical application.

The issue occurred approximately 1–2 times per day and 8–10 times per week, often requiring manual investigation and service restoration.

Average recovery time before automation was approximately 30–45 minutes.

## Solution

Developed a PowerShell automation workflow capable of:

- Detecting service outages.
- Restarting critical backend services.
- Restoring service dependencies.
- Generating automated recovery logs.
- Executing automatically through Windows Task Scheduler.

Additional functionality included:

- Error handling and exception management.
- Automated log folder creation.
- Service dependency restoration.
- Recovery success and failure logging.
